# Деплой бота РУМСЭТ на Railway

Эта инструкция описывает, как разместить бота на сервисе [Railway](https://railway.com) — облачной платформе, которая автоматически запускает ваш код на сервере с публичным URL.

> **Что такое деплой?**
> Деплой (от англ. deploy) — это размещение приложения на удалённом сервере, чтобы оно работало 24/7 и было доступно из интернета. Сейчас бот работает только на вашем компьютере; после деплоя он будет работать на сервере Railway.

---

## Что понадобится

1. **Аккаунт на GitHub** — [создать здесь](https://github.com/signup)
2. **Аккаунт на Railway** — [создать здесь](https://railway.com) (можно войти через GitHub)
3. **Проект загружен в GitHub-репозиторий** (см. раздел ниже)

---

## Шаг 1. Создание Git-репозитория

> **Что такое Git?**
> Git — система контроля версий. Она хранит историю изменений в коде. GitHub — это сайт, где хранятся Git-репозитории в интернете.

> **Где выполнять:** в терминале, внутри папки проекта.

### 1.1. Инициализация Git

```bash
git init
```

**Что делает:** создаёт пустой Git-репозиторий в текущей папке.
**Ожидаемый результат:** сообщение `Initialized empty Git repository`.

### 1.2. Добавление файлов

```bash
git add .
```

**Что делает:** добавляет все файлы проекта в Git (кроме перечисленных в `.gitignore` — туда входит `.env` с секретами).

### 1.3. Первый коммит

```bash
git commit -m "Initial commit: РУМСЭТ MAX bot"
```

**Что делает:** сохраняет текущее состояние проекта как точку в истории.

### 1.4. Создание репозитория на GitHub

1. Откройте [github.com/new](https://github.com/new).
2. Введите имя: `roomset-max-bot`.
3. Выберите **Private** (приватный).
4. Нажмите **Create repository**.
5. GitHub покажет команды — выполните те, что в разделе «push an existing repository»:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/roomset-max-bot.git
git branch -M main
git push -u origin main
```

**Ожидаемый результат:** код появился на странице вашего репозитория на GitHub.

## Шаг 2. Создание проекта на Railway

### 2.1. Вход в Railway

1. Откройте [railway.com](https://railway.com).
2. Войдите через GitHub.

### 2.2. Создание нового проекта

1. Нажмите **New Project**.
2. Выберите **Deploy from GitHub repo**.
3. Выберите репозиторий `roomset-max-bot`.
4. Railway автоматически определит, что это Node.js-проект, и начнёт сборку.

---

## Шаг 3. Настройка переменных окружения

> **Важно!** Без переменных окружения бот не запустится.

1. В дашборде Railway откройте ваш сервис (он появится как карточка).
2. Перейдите на вкладку **Variables**.
3. Добавьте следующие переменные:

| Переменная | Значение | Обязательно |
|---|---|---|
| `BOT_TOKEN` | Ваш токен бота MAX | Да |
| `PUBLIC_BASE_URL` | `https://ваш-сервис.up.railway.app` | Да |
| `WEBHOOK_PATH` | `/webhook/max` | Нет (по умолчанию) |
| `CATALOG_URL` | `https://room-set.ru/virtualnyj-shourum/` | Нет (по умолчанию) |
| `SECOND_MESSAGE_DELAY_MS` | `1500` | Нет (по умолчанию) |
| `LOG_LEVEL` | `info` | Нет (по умолчанию) |
| `NODE_ENV` | `production` | Нет (по умолчанию) |

> **Как узнать PUBLIC_BASE_URL?**
> После первого деплоя Railway присвоит вашему сервису домен. Найдите его на вкладке **Settings** → **Networking** → **Public Domain**. Если домена нет — нажмите **Generate Domain**. URL будет вида `https://roomset-max-bot-production.up.railway.app`.

> **Переменная `PORT`:** Railway задаёт `PORT` автоматически. **Не добавляйте** `PORT` вручную.

---

## Шаг 4. Настройка публичного домена

1. В Railway откройте ваш сервис.
2. Перейдите на **Settings** → **Networking**.
3. Нажмите **Generate Domain** (если домен ещё не создан).
4. Скопируйте полученный URL.
5. Вернитесь в **Variables** и установите `PUBLIC_BASE_URL` равным этому URL (с `https://`).

---

## Шаг 5. Проверка деплоя

### 5.1. Проверка health-check

Откройте в браузере: `https://ваш-домен.up.railway.app/health`

**Ожидаемый результат:**
```json
{ "status": "ok" }
```

### 5.2. Проверка логов

1. В Railway откройте ваш сервис.
2. Перейдите на вкладку **Deployments**.
3. Нажмите на последний деплой.
4. Перейдите на вкладку **Logs**.
5. Вы должны увидеть:
   ```
   roomset-max-bot v1.0.0 запускается
   Сервер запущен на порту ...
   Webhook успешно зарегистрирован в MAX
   ```

### 5.3. Проверка бота в MAX

1. Откройте MAX.
2. Найдите вашего бота или перейдите по ссылке `https://max.ru/roomset_bot`.
3. Нажмите «Начать».
4. Должно прийти приветственное сообщение с кнопкой.
5. Через 1–2 секунды должно прийти второе сообщение с контактами.

## Обновление после изменений

### Если вы изменили код

```bash
git add .
git commit -m "Описание изменений"
git push
```

Railway автоматически пересоберёт и перезапустит бота.

### Если вы изменили переменные окружения

Измените значение в Railway → **Variables**. Railway автоматически перезапустит сервис.

---

## Типичные проблемы при деплое

### Railway показывает ошибку сборки

- Откройте логи сборки (вкладка **Deployments** → нужный деплой → **Build Logs**).
- Чаще всего: забыли закоммитить какой-то файл. Проверьте `git status`.

### Бот запустился, но не отвечает

1. Проверьте, что `PUBLIC_BASE_URL` правильный (полный HTTPS URL).
2. Проверьте логи — есть ли ошибка регистрации webhook.
3. Убедитесь, что токен бота актуален.

### Ошибка `Cannot find module`

Убедитесь, что `package.json` и `tsconfig.json` закоммичены в Git:
```bash
git status
```

### Railway не определяет проект как Node.js

Убедитесь, что `package.json` лежит в корне репозитория (не в подпапке).

## Альтернативные платформы

Бот также можно задеплоить на:

- **[Render](https://render.com)** — аналогичный процесс, бесплатный план доступен.
- **VPS (любой Linux-сервер)** — установите Docker, скопируйте проект, запустите `docker compose up -d`.
- **[Fly.io](https://fly.io)** — поддерживает Dockerfile из коробки.

Во всех случаях главное:
1. Задать переменные окружения.
2. Обеспечить публичный HTTPS URL.
3. Убедиться, что `PUBLIC_BASE_URL` указывает на реальный адрес.
